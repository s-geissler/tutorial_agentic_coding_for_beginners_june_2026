Step 3 — the one-time setup (3 min)

Some of the engineering skills need to know where you keep things. Run this once per repo.

1

Run /setup-matt-pocock-skills inside opencode

It will quiz you — three or four short questions:

  • Which issue tracker do you use? (GitHub Issues / Linear / local markdown / none)
  • Which labels do you apply when triaging? (e.g. bug, enhancement, chore)
  • Where do you want domain docs saved? (e.g. docs/)

For the lecture, pick local markdown for the tracker (we're not wiring up GitHub), and accept the default labels. The setup writes a small AGENTS.md section so the other skills know where to put things.
