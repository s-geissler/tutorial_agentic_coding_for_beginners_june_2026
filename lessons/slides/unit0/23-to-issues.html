Step 7 — /to-issues decomposes the PRD (5 min)

A PRD is too big to be one task. This skill slices it into small, independently-grabbable issues — each one a vertical slice that ships value on its own.

1

Trigger

# inside opencode:
/to-issues

It will read the PRD you just generated, quiz you briefly on what counts as a slice, and produce a list of issues. Each one has:

  • a title in plain language
  • an acceptance criteria block
  • a label from your triage vocabulary
  • any dependencies on other issues

Since we picked "local markdown" for the tracker in step 4, the issues land as docs/issues/NNN-slug.md files. Commit them.
