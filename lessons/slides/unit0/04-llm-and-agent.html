The LLM and the agent

Two terms, two things. The LLM is the driver. The agent is what you get when you wrap an LLM in a harness.

LLM

The model itself

A next-token predictor. Nothing more. The same LLM powers ChatGPT, Claude Code, opencode, and your curl call to the API. The "brain in a box".

agent

LLM  +  harness

An LLM, plus the loop that feeds it context, the tools it can call, the memory it accumulates, and the UI you type into. The thing you actually interact with.

The LLM sits inside the agent. The harness is everything wrapped around it.

The next 30 minutes unpack one of those two things at a time — first the LLM, then the harness, then the loop that ties them together. By the end, you should be able to point at any agentic tool and say: "that's the LLM, and that's the harness".
