One folder = one slide = one <section>

Each slide lives in its own folder with self-contained HTML, CSS, and JavaScript.

slides/
├── 01-title/
│   ├── index.html      ← one <section class="slide">
│   ├── style.css        ← slide-local styles
│   └── script.js         ← slide-local behaviour
├── 02-philosophy/
├── …
└── 07-getting-started/
1

Data attributes drive navigation

data-section, data-title, and data-toc populate the TOC, progress bar, and section label.

2

The build extracts only the section

Your index.html is a full HTML document for standalone preview; the build extracts <section class="slide"> and inlines it into the bundle.
