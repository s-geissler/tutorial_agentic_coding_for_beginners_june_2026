Tokens, scaffolds, and templates keep decks consistent

A layered design system separates tokens from structure from content.

1

Design tokens (theme/tokens.css)

CSS custom properties for brand colors, surfaces, text, borders, code syntax, shadows, and typography. Light and dark variants included.

2

Base styles (theme/base.css)

Slide layout, typography, code blocks, lists, links, and responsive helpers. Pre-built widget classes include .cards, .step, .diagram, .compare, .takeaways, and more.

3

Scaffolds (template/scaffolds/{title,section,default,picture}/)

Pre-made slide starters that wire up the correct data-* attributes, header structure, and layout conventions. Use node deck.js add --scaffold=<name>.

4

Template shell (template/_template.html)

The outer HTML that wraps all slides: progress bar, TOC overlay, nav buttons, and theme toggle. Build replaces {{ placeholders }}.
