The deck-authoring skill turns an AI agent into a slide collaborator

This repo bundles a skill at .opencode/skills/deck-authoring/ that guides the Scientific Slide Designer agent through a disciplined five-phase workflow.

1

Absorb — read the design language

The agent reads DESIGN.md, inspects 2–3 existing slides, and skims the active theme/ and template/ before writing anything.

2

Plan — decide the narrative arc

Map content to scaffolds. Leave numeric gaps for inserts. State the plan before generating.

3

Scaffold — use deck.js add --scaffold

Four archetypes: title (cover), section (divider), default (content), picture (figure-led).

4

Author — write the slides

Prefer theme classes (.cards, .step, .diagram) over custom CSS. Never edit dist/. Use fractional prefixes for mid-deck inserts.

5

Build & verify

Run node deck.js build, open dist/index.html, check console and TOC. The agent does not hand-edit the bundle.

Included agent

The Scientific Slide Designer agent role is configured in this repo to enforce these rules. It produces publication-quality decks using the design tokens and scaffolds.

Key guardrails

The skill prevents common mistakes: no hand-editing dist/, no <base> tags, no manual renumbering, no CDN dependencies without approval.
