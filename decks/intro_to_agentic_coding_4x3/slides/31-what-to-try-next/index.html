What to try next

You've used five skills. There are twenty more in the collection. Here are four worth pulling on next.

1
Take a real bug to /diagnose

Find something broken in a project you actually care about. Run the diagnose skill. Notice the red-green discipline it imposes — the model is forbidden from guessing before reproducing.

2
Run /zoom-out on an unfamiliar file

Open a project you didn't write. Pick a file that scares you. Ask the agent to explain it in the context of the whole system. Compare to reading the file cold — the framing is the value.

3
Write a CONTEXT.md for a project you own

Pick a real codebase. Spend 30 minutes distilling its jargon into a one-page shared language. Watch the next session of opencode against that project become twice as productive.

4
Wire opencode against a local model

Install Ollama, pull a small model, point opencode at it via opencode.json. Notice the capability drop — and notice that for narrow tasks, the drop is smaller than you expected.
